## 🛠️ How I'm Using This Repo

1. **One GitHub Issue per week** — Copy template from [weekly-issues/](weekly-issues/) into a new issue.
2. **Daily commits** — Even small notes count. Commit message format: `Week X Day Y: <what I did>`
3. **Notes per topic** — One markdown file in [notes/](notes/) per concept.
4. **Code in public** — Every hands-on project lives in [code/](code/) with its own README.
5. **Sunday review** — Close the week's issue, update README progress, plan next week.

## 🧠 Study Method

- **Pomodoro**: 25 min focus + 5 min break
- **Feynman**: Write a 1-paragraph plain-English summary after each topic
- **Build > Read**: Spend ≥60% of time writing code, not just reading docs
- **Public learning**: Weekly LinkedIn/Twitter post about progress
