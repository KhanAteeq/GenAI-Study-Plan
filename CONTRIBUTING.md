# Contributing (to my own learning 😄)

This repo is a **personal study journal**, not a typical OSS project. The "contribution" rules below are the ones I hold _myself_ to.

## Branching
- `main` is the live journal. Direct commits OK for notes & docs.
- For each milestone project (`code/month-1-chatbot`, etc.), use a feature branch and PR into `main` so I practice PR workflow.

## Commit Messages
```
<area>: <what changed>

Examples:
notes: add week 3 SK basics
code(month-1): MathPlugin auto-call working
deliverable: week-04 chatbot demo
chore: update progress
```

## PR Checklist (for code projects)
- [ ] README updated (run instructions still accurate)
- [ ] No secrets in diff
- [ ] At least one test added or updated
- [ ] Issue linked (`Closes #N`)

## Issues
- Use the **Weekly Task** template for the 12 week milestones.
- Use the **Learning Note** template for stand-alone concept captures.
- Label every issue. Assign to the right milestone (Month 1/2/3).

## Definition of Done (per week)
- ✅ All checkboxes ticked _or_ explicitly deferred
- ✅ Code committed and runnable
- ✅ Notes file written
- ✅ Sunday review comment posted
- ✅ Issue closed
